# ROUTING_v3.0 - 技能路由规范

**默认路由器：production-router（工业级）**

## 路由器选择策略

| 场景 | 使用路由器 | 原因 |
|------|-----------|------|
| **默认/生产环境** | **production-router** | 工业级标准，带降级和监控 |
| 快速简单任务 | auto-selector | 极简，无额外开销 |
| 需要展示分析过程 | intelligent-router | 透明决策过程 |

## production-router 配置（默认启用）

**4维度评分算法：**
```
最终分数 = 意图识别(40%) + 复杂度评估(20%) + 上下文记忆(20%) + 技能成功率(20%)
```

**决策阈值：**
- 置信度 > 0.9：直接执行主技能
- 置信度 0.7-0.9：主技能 + 1个辅助技能
- 置信度 < 0.7：主技能 + 多个辅助技能 + 备选

**降级链：**
```
主技能失败 → 辅助技能1 → 辅助技能2 → ... → agent-tools（通用兜底）
```

**显示格式（强制）：**

## 性能目标
- 路由决策时间：< 10ms
- 技能选择准确率：> 98%
- 技能调用成功率：> 95%
- 平均任务完成时间：< 30秒

## 禁用路由的情况
以下情况**不使用自动路由**，直接指定技能：
- 用户明确要求"用 xxx 技能"
- 安全审查场景（强制使用 skill-vetter）
- 历史记忆搜索（强制使用 remembering-conversations）
- 文本人性化（强制使用 humanizer-zh）
