# 🔒 系统安全铁律 - 不可违反的操作规范

**生效日期**: 2026年3月8日  
**优先级**: 最高（绝对不可违反）  
**适用范围**: 所有Linux/Unix系统操作

---

## 第一条: rm 操作强制确认原则（绝对铁律）

### 🚨 核心规定

```
每次执行 Linux rm 命令（或任何删除操作）之前，
必须获得用户明确的书面/口头确认。

禁止条件:
❌ 禁止在没有用户确认的情况下执行 rm 命令
❌ 禁止使用 rm -rf 强制递归删除
❌ 禁止删除任何用户未明确授权的文件/目录
```

### 📋 执行流程

```
用户请求删除 → 系统确认意图 → 明确列出目标 → 用户二次确认 → 执行删除
     ↑                                                              ↓
     └──────────────── 任何一步用户取消，立即停止 ───────────────────┘
```

### ✅ 正确执行示例

```
用户: "删除这个文件"
系统: "确认删除 /path/to/file.txt 吗？此操作不可恢复。[是/否]"
用户: "是"
系统: "再次确认：文件 /path/to/file.txt 将被永久删除。确认？[是/否]"
用户: "是"
系统: "执行删除..."
```

### ❌ 禁止执行示例

```
用户: "帮我清理一下"
系统: "好的，正在删除..." ← 错误！未获得明确确认

或

系统直接执行 rm -rf /path ← 绝对禁止！
```

## 第二条: 数据保护原则

```
1. 优先使用 mv 代替 rm（移动到 trash/archives）
2. 重要文件删除前必须备份
3. 批量删除必须有清单确认
4. 系统关键文件（/etc, /usr等）绝对禁止删除
```

---

## 第三条: 高危命令白名单

### 以下命令必须获得明确授权

| 命令 | 风险等级 | 必须确认 |
|------|----------|----------|
| `rm` | 🔴 极高 | ✅ 必须 |
| `rm -rf` | 🔴🔴🔴 致命 | ✅ 必须 |
| `dd` | 🔴 极高 | ✅ 必须 |
| `mkfs` | 🔴 极高 | ✅ 必须 |
| `>` 重定向覆盖 | 🟡 高 | ✅ 必须 |
| `chmod -R` | 🟡 高 | 建议确认 |
| `chown -R` | 🟡 高 | 建议确认 |

---

## 第四条: 例外情况

### 以下情况可执行删除（仍需记录）

```
✅ 用户明确编写的脚本中包含 rm 命令
✅ 临时文件清理（/tmp目录，且文件创建时间>7天）
✅ 用户明确指定的垃圾文件（如 .log.old）
⚠️ 但仍需记录日志
```

---

## 违规处理

### 如果违反此铁律

```
1. 立即停止当前操作
2. 向用户报告违规情况
3. 恢复被删除文件（如果可能）
4. 记录违规日志到 .learnings/ERRORS.md
5. 等待用户进一步指示
```
